## 2. LE CONCEPT CENTRAL

### ivoire.io = La plateforme "B2B2C" qui centralise 3 choses :

```
┌─────────────────────────────────────────────────┐
│                   IVOIRE.IO                      │
│          "The Digital Gateway to                 │
│             Ivory Coast"                         │
├─────────────┬───────────────┬───────────────────┤
│  🧑‍💻 TALENTS  │  🚀 STARTUPS  │  🏢 SERVICES     │
│             │               │                   │
│ Portfolios  │ Annuaire      │ Santé             │
│ Sous-domaines│ Product Hunt  │ Logement          │
│ CV & Projets│ local         │ Éducation         │
│ Marketplace │ Levées fonds  │ Emploi            │
│ hire        │ Vote/Ranking  │ Data/API          │
└─────────────┴───────────────┴───────────────────┘
```

### Le principe "Vendeur de pioches" :
> Tu ne construis pas des apps concurrentes. Tu construis **l'infrastructure** sur laquelle les autres construisent. C'est la stratégie Stripe/GitHub appliquée à un pays.

### Les 2 types de sous-domaines :

| Type | Exemple | Objectif | Prix |
|------|---------|----------|------|
| **Personnel** | ulrich.ivoire.io | Portfolio dev, viralité, communauté | Gratuit |
| **Entreprise** | agence-x.ivoire.io | Label de confiance, visibilité pro | Payant (Premium) |

**Stratégie** : Commencer par les sous-domaines personnels (gratuit) → crée l'effet réseau → les entreprises payent pour rejoindre l'écosystème.

## 3. ARCHITECTURE PRODUIT

### Structure des portails :

```
ivoire.io                    ← Page d'accueil / Manifeste
├── devs.ivoire.io           ← Annuaire développeurs
├── [nom].ivoire.io          ← Portfolios individuels
├── startups.ivoire.io       ← Product Hunt ivoirien
├── jobs.ivoire.io           ← Offres d'emploi tech
├── learn.ivoire.io          ← Éducation / Concours
├── health.ivoire.io         ← S.O.S Santé intégré
├── data.ivoire.io           ← Open data / API
├── events.ivoire.io         ← Hackathons & Conférences
├── invest.ivoire.io         ← Matching startups/investisseurs
└── api.ivoire.io            ← API Gateway (données écosystème)
```

### Système d'authentification : "Ivoire ID"
- Un compte unique pour tout l'écosystème (SSO)
- Connexion via email, téléphone, ou GitHub/LinkedIn
- L'utilisateur accède à tous les portails avec un seul login
- Préparation pour le futur : "Se connecter avec Ivoire ID" (comme Google/Apple Sign-In)

## 5. STACK TECHNIQUE

### Choix validés par les 3 analyses :

| Composant | Technologie | Pourquoi |
|-----------|-------------|----------|
| **Frontend** | **Next.js** | SEO agressif, SSR, performance |
| **Mobile/Dashboard** | **Flutter Web** | Dashboards interactifs, cross-platform |
| **Backend / Auth / DB** | **Supabase** (PostgreSQL) | Auth, real-time, API auto, gratuit au début |
| **DNS / CDN / Sécurité** | **Cloudflare** | Wildcard DNS, SSL auto, Workers pour routing |
| **Hébergement** | **Vercel** | Déploiement Next.js natif, edge functions |
| **IA** | **API Gemini / OpenAI** | Génération automatique descriptions, modération |
| **Email** | **Resend** ou **Mailgun** | Emails transactionnels |
| **Analytics** | **Plausible** ou **PostHog** | Analytics respectueux de la vie privée |

### Architecture Wildcard Subdomains :
```
Requête HTTP → Cloudflare DNS (*.ivoire.io)
    → Vercel Edge Function
        → Lit le subdomain depuis le header Host
        → Query Supabase : SELECT * FROM profiles WHERE slug = subdomain
        → Rendu du portfolio/page correspondant
```

### 📌 PHASE 2 — MVP "HUB DES TALENTS" (Jours 11-45)
**Objectif : Rendre ivoire.io viral via les sous-domaines**

#### A. Générateur de Portfolio (nom.ivoire.io)
- L'utilisateur s'inscrit → choisit son slug → connecte GitHub/LinkedIn
- Génération automatique d'une page portfolio :
  - Photo + bio
  - Compétences & technologies
  - Projets (importés depuis GitHub)
  - Liens (LinkedIn, Twitter, site perso)
  - Bouton "Me contacter" / "M'embaucher"
- Template unique au début, personnalisable plus tard
- **Effet viral** : chaque dev partage son lien → marketing gratuit

#### B. Annuaire des Talents (devs.ivoire.io)
- Liste publique de tous les développeurs inscrits
- Filtres : techno, ville, disponibilité, expérience
- Recherche full-text
- Classement par popularité / récence

#### C. Base de données Supabase :
```sql
-- Tables principales
profiles (
  id, slug, full_name, email, bio, avatar_url,
  github_url, linkedin_url, skills[], city,
  is_available, type, created_at
)

subdomains (
  id, slug, profile_id, type, is_active, created_at
)

waitlist (
  id, email, name, desired_slug, type, created_at
)
```

## 9. RISQUES & PARADES

| Risque | Impact | Parade |
|--------|--------|--------|
| **Pas d'adoption** | Critique | Validation par waitlist AVANT de coder |
| **Spam / contenu bas de gamme** | Élevé | Modération IA + curation manuelle stricte |
| **Conflit employeur** | Moyen | Vérifier clause de non-concurrence, lancer sous marque (pas nom perso) |
| **Copie du concept** | Faible | L'avantage premier entrant + le domaine = barrière à l'entrée |
| **Surcharge technique** | Moyen | MVP module par module, jamais tout en même temps |
| **Coûts serveur** | Faible | Stack gratuite/freemium au début (Supabase, Vercel, Cloudflare) |
